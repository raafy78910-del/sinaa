# دليل الإعداد — نظام شؤون الموظفين (مجاني بالكامل)

التركيبة:

```
المتصفح (index.html)  ⇄  Google Apps Script (Code.gs = الخادم/API)  ⇄  Google Sheet (قاعدة البيانات)
```

---

## ⚠️ إذا ظهرت رسالة "كلمة المرور خطأ"

أكبر سبب شائع: الملفين **Code.gs** و **index.html** لازم يكونوا من نفس الإصدار (لأن كلمات المرور صارت تُشفّر الآن قبل ما تُرسل — لو عندك نسخة قديمة من أحدهما متل واحدة بدون تشفير، ما رح تتطابق). الحل الأضمن:

1. خذ نسخة **Code.gs** المرفقة هلق (الأحدث) والصقها بمحرر Apps Script (احذف القديم بالكامل أولاً).
2. شغّل دالة `setupSheets` من جديد (هاد بيعيد إنشاء شيت `Users` بكلمة مرور admin123 المشفّرة بالطريقة الصحيحة). لاحظ إنه هذا بيمسح أي بيانات سابقة بكل الشيتات — إذا عندك بيانات حقيقية لا تخسرها، خذ نسخة احتياطية من الشيت قبل التشغيل.
3. **Deploy ← Manage deployments ← ✏️ تعديل ← Version: New version ← Deploy** (لازم نسخة جديدة عشان التعديلات تشتغل).
4. خذ نسخة **index.html** المرفقة هلق وافتحها، اربطها بنفس رابط النشر.
5. سجّل دخول بـ `admin` / `admin123`.

لو استمرت المشكلة، تأكد إنه:
- ما في مسافات زائدة عند نسخ الرابط أو كلمة المرور.
- النشر فعليًا بصلاحية "Who has access: Anyone".
- شيت `Users` فيه صف بيانات فعلي (مو بس الصف الأول للعناوين).

## الخطوة 1 — إنشاء قاعدة البيانات

1. افتح [sheets.google.com](https://sheets.google.com) وأنشئ ملف جديد فارغ.
2. سمّيه مثلاً: **قاعدة بيانات شؤون الموظفين**.

## الخطوة 2 — لصق كود الخادم

1. من القائمة: **الإضافات (Extensions) ← Apps Script**.
2. احذف أي كود موجود بالافتراض، وافتح ملف `Code.gs` المرفق وانسخ محتواه بالكامل، والصقه.
3. احفظ المشروع (Ctrl+S).

## الخطوة 3 — إنشاء الجداول تلقائيًا

1. من القائمة المنسدلة بأعلى المحرر اختر الدالة **`setupSheets`** ثم اضغط ▶ **تشغيل**.
2. أول مرة، وافق على صلاحيات Google (Authorize) — اختر "Advanced ← Go to project (unsafe)" إذا ظهر تحذير (طبيعي لأي سكربت شخصي).
3. بعد النجاح رح تظهر رسالة: اسم المستخدم `admin` وكلمة المرور `admin123`.
4. تأكد إنه انعملت الأوراق: `Users`, `Employees`, `Formations`, `Leaves`, `Transfers`, `AssignmentsViolations`, `Logs`.

## الخطوة 4 — نشر الخادم كتطبيق ويب

1. **Deploy ← New deployment ← ⚙ Web app**.
2. **Execute as**: Me | **Who has access**: Anyone.
3. اضغط **Deploy**، وافق على الصلاحيات، وانسخ **Web app URL**.

> ⚠️ كل تعديل على `Code.gs` يتطلب **Deploy ← Manage deployments ← ✏️ ← New version ← Deploy** عشان ينعكس.

## الخطوة 5 — ربط الواجهة

افتح `index.html` بمحرر نصوص، دوّر على:
```js
var DEFAULT_API_URL = '';
```
وحط الرابط جوه القوسين، احفظ. أو الصقه مباشرة بشاشة "ربط والمتابعة" عند فتح الملف (بس رح تحتاج تعيد لصقه كل مرة تفتح الصفحة من جديد).

## الخطوة 6 — أول تسجيل دخول

سجّل دخول بـ `admin` / `admin123`، وغيّرها من **إدارة المستخدمين**، وأضف باقي الحسابات (سكرتير، اطّلاع...).

## الخطوة 7 — استضافة الواجهة أونلاين (اختياري، مجاني)

ارفع `index.html` على **GitHub Pages** أو **Netlify / Vercel** (الخطط المجانية) للحصول على رابط ثابت.

## الوحدات المتوفرة

| الوحدة | الوصف |
|---|---|
| بيانات الموظفين | سجل كل موظف (هوية، وظيفة، مبحث، حالة...) |
| بيانات التشكيلات | سجل حصص كل معلم (معلم + صف/شعبة + مبحث + عدد حصص)، أو "تفريغ" لنشاط غير صفي (مختبر، كشافة...) |
| **جدول التشكيلات** | عرض تلقائي لنفس بيانات التشكيلات لكن كمصفوفة (معلم × شعبة)، بنفس شكل ملف "التشكيلات المقترحة" — مع عمود "التفريغ" وعمود "المجموع" وصف "مجموع الحصص" بالأسفل، مرتّبة حسب الصف ثم رمز الشعبة (أ، ب، ج...) |
| إجازات الموظفين | تسجيل ومتابعة الإجازات |
| **طلبات المواصلات** | نموذج بدل المواصلات الرسمي — اختيار الموظف من بيانات الموظفين المُدخلة مسبقًا (مع تعبئة تلقائية لرقم الهوية والوظيفة)، جدول خط سفر متعدد الأسطر (من/إلى/المسافة/السعر) قابل للإضافة والحذف، ومجموع يُحسب تلقائيًا. زر "طباعة النموذج الرسمي" بكل سطر يصدّر التقرير **بنفس تنسيق نموذج وزارة التربية والتعليم المرفق** (الترويسة الرسمية بشعار الوزارة، كل الحقول والتسلسل بنفس الترتيب، وأسطر فارغة لتوقيعات المسؤول المباشر/مسؤول الوحدة/قسم الرواتب/مدير العلاوات/مدير الشؤون الإدارية، ورقم النموذج بالتذييل) |
| النقل والتقاعد والاستقالة | إجراءات نقل/تقاعد/استقالة الموظفين |
| التكليفات والمخالفات | تكليف الموظفين بمهام، وتسجيل المخالفات |
| التقارير | تصدير أي وحدة (بما فيها جدول التشكيلات) كملف Excel، أو طباعتها/حفظها PDF بترويسة رسمية |
| إدارة المستخدمين | للمدير فقط — إنشاء حسابات وتحديد الصلاحيات |

### كيف تُدخل بيانات التشكيلات بشكل صحيح؟

كل سطر بشاشة "بيانات التشكيلات" يمثّل **حصة معلم واحد بشعبة واحدة**. مثلاً المعلم "محمد أحمد خليل" يعلّم رياضيات بـ 8أ و8ب، يعني سطرين منفصلين:
- محمد أحمد خليل | حصة صفية | 8أ | رياضيات | 5 حصص
- محمد أحمد خليل | حصة صفية | 8ب | رياضيات | 5 حصص

ولو عنده تفريغ (إشراف مختبر، نشاط...)، سطر إضافي بنوع "تفريغ" بدون صف/شعبة:
- محمد أحمد خليل | تفريغ | — | مختبر علوم | 2 حصص

شاشة **"جدول التشكيلات"** بتجمع هاي السطور تلقائيًا بمصفوفة واحدة لكل معلم، وبتحسب المجموع.

## الصلاحيات (حسب الدور)

| الوحدة | مدير | سكرتير | اطّلاع |
|---|---|---|---|
| الموظفين / التشكيلات / الإجازات / المواصلات / النقل | عرض + إضافة + تعديل + حذف | عرض + إضافة + تعديل (بدون حذف) | عرض فقط |
| التكليفات والمخالفات | عرض + إضافة + تعديل + حذف | عرض فقط | عرض فقط |
| إدارة المستخدمين | كاملة | لا صلاحية | لا صلاحية |

> ملاحظة عن وحدة المواصلات: شعار الوزارة بالترويسة مُستخرج مباشرة من ملف `نموذج_مواصلات.rtf` المرفق (مضمّن داخل `index.html` كصورة)، فما في حاجة لرفع أي ملف شعار إضافي — التقرير المطبوع يطلع بنفس الترويسة تلقائيًا.

التحقق يتم بالواجهة **وبالخادم معًا** (Code.gs يرفض أي طلب لا يطابق دور المستخدم)، مو بس بإخفاء الأزرار.

## الأمان

- **كلمات المرور لا تُخزَّن ولا تُرسَل كنص عادي** — يتم تشفيرها (SHA-256) بالمتصفح قبل إرسالها، والخادم يقارن التشفير فقط.
- هاد تحسين حقيقي، لكنه ليس بمستوى حماية إنتاجي كامل (لا يوجد "ملح" عشوائي لكل مستخدم، ولا قفل بعد محاولات فاشلة متكررة). لبيانات حساسة فعلية يُنصح لاحقًا بالانتقال لنظام مصادقة حقيقي (Google Identity).
- لا تشارك رابط النشر (Web app URL) مع غير الموثوقين.
- حدود الاستخدام المجاني لـ Apps Script سخية وتكفي مدرسة واحدة بسهولة — [التفاصيل هون](https://developers.google.com/apps-script/guides/services/quotas).

## شو الخطوة الجاية؟

- تصدير "جدول التشكيلات" مباشرة بصيغة قريبة جدًا من ملف Excel الأصلي (ترويسة المدرسة، توقيع المدير...).
- صلاحية "تعديل الملف الشخصي / كلمة المرور" لكل مستخدم بنفسه.
- ربط شاشة "النقل" تلقائيًا ببيانات الموظف (تحديث حالته الوظيفية تلقائيًا عند تنفيذ نقل/تقاعد/استقالة).

قول لي وين بدك نكمل.
